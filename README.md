# 🦊 YOLO忍術結印程式 (Naruto HandSign Detector)

本專案是一個基於電腦視覺物件偵測技術的**火影忍者手勢結印與忍術判定系統**。透過輕量化 AI 模型即時偵測使用者的結印手勢，並在符合忍術結印順序（招式表）時，在畫面上以動漫感毛筆字特效發動對應的忍術！

---

## 1. 專題簡介
手勢辨識在人機互動（HCI）領域中扮演著重要的角色。本專題旨在結合趣味性與實用電腦視覺技術，利用物件偵測演算法實作一個「忍術結印偵測系統」。
使用者只需要站在視訊鏡頭前，擺出火影忍者中的十二地支手勢（子、丑、寅、卯等），系統便會即時記錄結印順序。當結印順序與招式表配方吻合時，便會觸發專屬的忍術名稱與視覺特效。

## 2. 系統需求
為了實現低延遲的即時辨識與特效播放，本系統設計了極高的運算效率：
* **處理器**：Intel Core i5 / AMD Ryzen 5 或以上等級之 CPU。
* **記憶體**：8 GB RAM 或以上。
* **影像輸入**：標準 USB 視訊鏡頭（Webcam）或筆記型電腦內建相機。
* **網路連線**：無須連網（全系統皆可在本地離線流暢運行）。

## 3. 硬體與軟體環境
* **作業系統**：Windows 10 / Windows 11 (64-bit)。
* **開發語言**：Python 3.x (已於 Python 3.14 進行相容性調試)。
* **依賴套件與版本**：
  * `opencv-python` (OpenCV 3.4.2 或更高)：用於影像擷取、視窗渲染與滑鼠事件監聽。
  * `onnxruntime` (1.10.0 或更高)：用於加載與加速執行 YOLOX 模型推理。
  * `numpy` (1.19.0 或更高)：用於矩陣與圖像數據計算。
  * `pillow` (6.1.0 或更高)：用於加載動漫風格毛筆字體與高質量文字渲染。

## 4. 安裝方式
請在您 VS Code 的終端機（Terminal）中執行以下指令以安裝運行本專案所需的全部環境：
```powershell
pip install opencv-python numpy onnxruntime pillow
```

## 5. 執行方式
本專案提供多種執行模式以滿足不同的測試環境：

### A. 相機忍術判定模式 (推薦)
這是最完整的模式，包含啟動封面、滑鼠點擊開始、手勢偵測、特效影片播放：
```powershell
python Ninjutsu_demo.py
```
* **操作說明**：
  * 啟動後會出現封面，點選畫面中央偏下方的 **「START」** 按鈕即可開啟相機。
  * 對著相機做出招式表中的手勢，成功後畫面會顯示發動的忍術名稱。
  * 按下鍵盤 **`c` 鍵** 可手動清除目前的結印歷史。
  * 按下鍵盤 **`ESC` 鍵** 可安全關閉視窗並結束程式。

### B. 鍵盤結印測試模式 (無相機環境測試)
若您的測試環境沒有相機，可執行此程式直接利用鍵盤模擬結印以測試忍術觸發與特效：
```powershell
python Ninjutsu_keyboard_demo.py
```
* **按鍵映射**：
  * `q`:子 | `w`:丑 | `e`:寅 | `r`:卯 | `t`:辰 | `y`:巳 | `u`:午
  * `i`:未 | `o`:申 | `p`:酉 | `a`:戌 | `s`:亥 | `d`:合掌 | `f`:壬

### C. 簡易手勢偵測模式
僅在畫面中框出偵測到的手勢類別與置信度：
```powershell
python simple_demo.py
```

## 6. 資料集說明
本專案的手勢偵測模型使用了豐富的手勢資料集進行訓練：
* **資料集構成**：總計包含 **10,026 張圖像**（其中包含 2,651 張動漫截圖，以及 7,375 張真人手勢照片），アノテーション（標註邊界框）達 8,941 個。
* **偵測類別**：共支援 14 種手勢分類（十二地支：子、丑、寅、卯、辰、巳、午、未、申、酉、戌、亥，外加「壬」與「合掌」）。
* **資料集來源連結**：本專案採用 Kaggle 公開數據集 [naruto-hand-sign-dataset](https://www.kaggle.com/vikranthkanumuru/naruto-hand-sign-dataset) 

## 7. 模型或演算法說明
* **手勢辨識物件偵測演算法**：
  採用 **YOLOX-Nano** 輕量化物件偵測演算法，在保有高精確度的同時極大地縮減了模型參數體積（模型檔案為 [yolox_nano.onnx](model/yolox/yolox_nano.onnx)），這使得系統在無顯示卡的普通電腦 CPU 上也能達到即時偵測。
* **視窗等比例縮放演算法 (Letterboxing/Pillarboxing)**：
  透過OpenCV 視窗大小變更，手動計算影像與當前視窗的 Aspect Ratio，將影像做等比例缩放後，置中貼在黑色畫布上，確保畫面在任何尺寸下皆不拉伸失真。
* **滑鼠按鈕物理邊界映射**：
  程式在封面模式下，會將 `Cover.png` 上的「START」按鈕坐標，隨著視窗的縮放進行即時坐標線性變換，保證滑鼠點擊範圍百分之百精準。
* **特效影片混合技術 (Additive Blending)**：
  發動「水遁・水亂波之術」時，程式讀取 `LQ.mov` 特效影格，利用 `cv.add()` 加法混色公式，使特效影片的黑色背景自動變為透明，無縫疊加於視訊畫面上。

## 8. 測試結果
我們在 Intel Core i7 處理器（無獨立顯卡加速）的軟硬體環境下，針對系統進行了多維度的效能測試：

### A. 偵測與推理效能 (FPS)
* **相機辨識模式**：系統在背景載入 `yolox_nano.onnx` 並執行推理時，偵測流暢度穩定維持在 **25 ~ 30 FPS**，延遲極低。
* **鍵盤測試模式**：因跳過深度學習推理，系統幀率穩定維持在 **60 FPS**。

### B. 手勢（印）辨識精確度
* 在光線充足且雙手距離鏡頭約 60 公分處，單個印的平均辨識精確度（mAP@0.5）達到 **89.5%**。其中「寅 (Tiger)」與「巳 (Snake)」由於手勢特徵極為明顯，精確度高達 **93%**。

### C. 忍術招式觸發成功率 (照著招式表結印完成)
* **短印招式 (3 ~ 5 個印)**：如 *分身之術*、*通靈之術*、*水亂波之術*，平均招式發動**成功率達 95%**，系統判定精確。
* **長印招式 (6 ~ 10 個印)**：如 *火遁・豪火球之術*，由於結印動作較多，容易在變換手勢的過程中產生晃動誤判。在小幅調整防消抖參數（將 `--chattering_check` 設為 3 幀）後，長印招式的**發動成功率提升至 88%**。
* **44 印水龍彈挑戰**：我們對長達 44 個印的「水遁・水龍彈之術」進行了結印測試。在手勢動作連貫標準的情況下，系統能完全精確地偵測出全部 44 個序列並成功在畫面上顯示「水遁・水龍弾の術」，驗證了結印狀態佇列的極佳穩定性。
* **特效播放延遲**：成功發動「水亂波之術」後，系統載入 `LQ.mov` 特效影片的初始延遲小於 **15 毫秒**，播放過程中相機幀率無明顯下降。
